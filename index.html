What Inside of Uniprime & How to configure

How to configure & Install Packages

When you get Uniprime.zip file form Themeforest first needt to unzip the file and run few commands following:

Uniprime run

  1. yarn install on Uniprime(root folder) folder.
  2. yarn start run server

Uniprime run in Next.js server

  1. yarn dev on next-landing

Then , please go to address localhost:3000 on your browser and You will find default landing page.
Similarly
localhost:3000/classic for App Classic Landing Page
localhost:3000/modern for App Modern Landing Page
localhost:3000/blog-grid for Blog-Grid Page
localhost:3000/blog-list for Blog-List Page
localhost:3000/blog-single for Blog-Single Page

Uniprime run in Gatsby.js server

  1. yarn dev gatsby-landing

Then , please go to address localhost:8000 on your browser and You will find default landing page.
Similarly
localhost:8000/classic for App Classic Landing Page
localhost:8000/modern for App Modern Landing Page
localhost:8000/blog-grid for Blog-Grid Page
localhost:8000/blog-list for Blog-List Page
localhost:8000/blog-single for Blog-Single Page

Folder Structure

Go to Uniprime --> packages folder . After entering to packages folder, You will find some other folders.

  1. common
  2. next-landing
  3. gatsby-landing

src folder

The whole work is in packages -> common/src folder . After entering to common/src folder, You will find:

  1. assets
  2. reusecore
  3. sections
  4. sections-gatsby
  5. theme

Now, we will discuss about each and every folders and their tasks .

assets

In assets folder, you will find the data and images folder. In data folder, you will find all of the data uses in this template. You will also find the image folder, where all of the images are kept on the basis of the specific page.

reusecore

reusecore folder is most importantly use, you will find some basic components like Layout, Button, Counter, Navigation etc. We have written these components to make the developer's life easy. Here in details:
Accordion -> By this you can make any Accordion with your own style
Button -> By this you can make any Button with your own style
Counter -> By this you can make any Counter with your own style
Layout -> Here in Layout all grid system are included
Navigation -> By this you can make Navigation with your own style
PageHeader -> By this you can make BreadCrumb with your own style
Particle -> By this you can use Particle in any secction of your template
SectionTitle -> By this you can make any SectionTitle with your own style in template
SlickSlider -> By this you can make any slider with your own style in template
VintageBox -> This one is for adding any vintage with any section or elements
By using these basic components, you can write custom components according to your need.

sections

Here all the section include that’s we are used in our template like banner, about, features, news, pricing etc etc for nextjs version.

sections-gatsby

Here all the section include that’s we are used in our template like banner, about, features, news, pricing etc etc for gatsby version.

theme

Here all Theming related (mainly color) functions are includded,
app -> For app landing default and modern theming
classic -> For app landing classic theming
blog -> For all blog related theming

next-landing folder

In this folder , you will find the pages folder of next js and next configuration files. If you want to change something in next js folder , change here .

pages

As we have used next.js , we have a script to your package.json like this: 

						{
						“scripts”: {
							“dev”: “next”,
							“build”: “next build”,
							“start”: “next start”
							}
						}

After that, the file-system is the main API. Every .js file becomes a route that gets automatically processed and rendered.
Please have a look at this link https://nextjs.org/docs/ for a quick look. You will understand the basic things so quickly.
As we have Three landing pages and three blog pages named index, classic, modern, blog-grid, blog-list, blog-single so we have six js files named: index.js for Default Landing Page.
classic.js for Classic Landing Page.
modern.js for Modern Landing Page.
blog-grid.js for Blog Grid Page.
blog-list.js for Blog List Page.
blog-single for Blog-Single Page .

gatsby-landing folder

According to gatsby configurations , we gave support for styled components and web fonts for our project. The other settings are as same as gatsby default configurations . You can follow the config documentations of gatsby Js "https://www.gatsbyjs.org/docs/customization/#customization .

Uniprime deployment process

Firebase Deployment for Gatsbay

You can follow this doc for firebase hosting https://firebase.google.com/docs/hosting/quickstart. We followed the documentation and integrated with our settings.

Step 1 : Install the Firebase CLI

  1. Install Node.js using one of the following options. Installing Node.js automatically installs npm. (Note: The Firebase CLI requires Node.js v6.0.0 or later.)
  2. Install the Firebase CLI using npm by running:
    npm install -g firebase-tools
  3. After installing the CLI, you must authenticate. Then you can confirm authentication by listing your Firebase projects. Sign into Firebase using your Google account by running the following command:
    firebase login --interactive
  4. Test that the CLI is properly installed and accessing your account by listing your Firebase projects. Run the following command:
    firebase projects:list

  5. Initialize your project

    To connect your local project to your Firebase project, run the following command from the root of your local project directory:
    firebase init
    Select Firebase Hosting and Firebase function from the list. Then you have to answer couple of questions like: What language would you like to use to write Cloud Functions? JavaScript Do you want to use ESLint to catch probable bugs and enforce style? No Do you want to install dependencies with npm now? No What do you want to use as your public directory? public Then, an index.html file will created under the public directory.

  6. Deploy your project

    To deploy your project, you have to do couple of things. At first, go to firebase.json file. Then delete all codes. Go to gatsby.firebase.json file, copy all of the codes and paste it in firebase.json file. Before running the below deploy command, you have to delete the public->index.html file. and go to landing->package.json, find engines and do the following "engines": { "node": "8" } or "engines": { "node": "10" } Then, To deploy to your site, run the following command from the root of your local project directory:
    for Gatsbyjs template: yarn gatsby-firebase-deploy

Now Deployment for Nextjs

  1. Install the Now CLI

    Install Now Desktop https://zeit.co/download or Now CLI https://zeit.co/download#now-cli

  2. Create now.json file

    If you want to host the next js project, Create a file named now.json in the root folder of your project. Then go to next.now.json file, copy all of the codes and paste it in now.json file.

  3. Deploy your project

    Finally, Run the following command to deploy it:
    now

    You may have to confirm your email address for first time. Please provide your email address in your command line and a confirmation email will be sent in your email address. Verify the email address and then run ‘now’ command again.
